# CLAUDE.md

Project-specific instructions for Claude Code.

## Repository Structure

This is the **get-shit-done** framework - a meta-prompting system for Claude Code and OpenCode.

### Critical: Source vs Install Directories

**DO NOT confuse source directories with install destinations!**

| Directory | Purpose | DO NOT EDIT DIRECTLY |
|-----------|---------|---------------------|
| `.claude/` | **LOCAL INSTALL** for Claude Code | Yes - generated by installer |
| `.opencode/` | **LOCAL INSTALL** for OpenCode | Yes - generated by installer |

**Source directories (edit these):**

| Directory | Purpose |
|-----------|---------|
| `commands/gsd/` | GSD command source files (Claude Code format) |
| `commands/opencode/` | GSD command source files (OpenCode format) |
| `get-shit-done/` | Shared references, templates, workflows |
| `agents/` | Subagent definitions |
| `opencode/` | OpenCode-specific source files (plugin, tool) |
| `bin/install.js` | Unified installer for both agents |

### How Installation Works

`bin/install.js` copies source files to install destinations:

**Claude Code:**
- Global: `~/.claude/`
- Local: `./.claude/`

**OpenCode:**
- Global: `~/.config/opencode/`
- Local: `./.opencode/`

### When Adding OpenCode Features

1. Create source files in `opencode/plugin/` or `opencode/tool/`
2. Update `bin/install.js` to copy them to the destination
3. Never edit `.opencode/` directly - it gets overwritten on install

### Commands

```bash
# Install globally (Claude Code)
node bin/install.js --global

# Install locally (current project)
node bin/install.js --local

# Interactive install (choose agents)
node bin/install.js
```
